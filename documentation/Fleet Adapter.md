## Что из себя представляет?
`Fleet Adapter` - это мост между роботами и системой Open-RMF, который распознает информацию от каждого робота индивидуально и передает это все Open-RMF. А также сам получает информацию от Open-RMF для того, чтобы обработать ее в понятный для робота язык и отправить ему.

---
#### Полноценную информацию по `Fleet Adapter` можете прочесть на официальной странице документации - [`Fleet Adapter Tutorial`](https://osrf.github.io/ros2multirobotbook/integration_fleets_adapter_tutorial.html#fleet-adapter-tutorial-python) 
---
## Что требуется знать для работы с Rombus?
- #### Один адаптер — один флот.
  Каждый адаптер рассчитан на неопределенное количество *одинаковых* роботов.
- #### Конфигурационный файл для адаптера. 
  Каждый адаптер требует наличие файла конфигурации, в котором указаны данные о роботах (скорость, масса, батарея и т.д.). 
- #### Файл графов для адаптера.
  Каждый адаптер требует наличие файла графов, в котором указаны точки и пути, по которым роботы могут передвигаться.
- #### В Rombus только 1 тип роботов (`turtlebot 3`).
  В действующей версии приложения существует только 1 тип роботов с которым можно взаимодействовать. Но тем не менее они имеют урезанную возможность для конфигурации.
- #### В Rombus только 2 типа конфигурационных файлов для адаптера (`turtlebot3_fleet_config.yaml`, `fast_turtlebot3_fleet_config.yaml`).
  Несмотря на то, что тип роботов один, они сконфигурированы с немного разной скоростью.

## Как запустить адаптер?
1. Запустите скрипт и проделайте все действия, которые описаны в [основном руководстве](Документация/Документация)
2. Выберите запуск адаптера
3. Введите название своей созданной карты
4. Введите адрес для отображения в [RMF Web](Документация/RMF Web) (Изначально `ws://localhost:8000/_internal`)
5. Введите полное название конфигурационного файла, который подходит именно для вашего флота роботов (сейчас доступны только `turtlebot3_fleet_config.yaml` и `turtlebot3_fleet_config.yaml`)
6. Введите полное название файла графов. Название формируется из того, какой номер графа был выбран вами при создании навигационных линий на карте в [traffic-editor](Документация/Traffic-editor) (пункт 11). Корректный формат ввода - `<Номер графа из traffic-editor>.yaml` (`0.yaml`, `1.yaml`и т.д.)
